AI coding agent를 실무에 쓰면서 가장 답답했던 건
**세션이 끝나면 맥락도 같이 사라진다는 점**이었습니다.

어제 알려준 규칙을 오늘 또 설명해야 하고,
왜 그런 결정을 했는지도 다시 꺼내야 했습니다.

그래서 **AI Barracks**라는 작은 오픈소스를 만들었습니다.
핵심 아이디어는 단순합니다.

**에이전트의 기억을 프롬프트가 아니라 Git 저장소에 남기자.**

프로젝트 디렉토리 안에 마크다운 파일로:

- 세션마다 무슨 작업을 했고 어떤 결정을 내렸는지 (영구 기록)
- 반복되는 교정을 규칙으로 축적 (다음 세션에 자동 적용)
- 프로젝트 지식을 토픽별로 정리 (필요한 것만 선택 로딩)

을 저장하고, 전부 Git으로 관리합니다.
DB도, 서버도, SaaS도 없습니다.

설계하면서 중요하게 본 건 두 가지였습니다.

**1. LLM이 잘 따르길 기대하지 않기**
세션 등록이나 정리 같은 작업은 CLI Hook으로 자동화했습니다.
LLM이 프로토콜을 잊어도 시스템은 정상 동작합니다.

**2. 특정 CLI에 묶이지 않기**
Claude Code, Gemini CLI, Codex CLI가 같은 프로젝트 컨텍스트를 공유합니다.
Claude가 rate limit에 걸리면 Gemini로 세션을 이어받는 것도 가능합니다.

**3. 규칙을 말하지 말고, 기계적으로 강제하기** (v0.9.0)
OpenAI Harness Engineering에서 배운 원칙입니다.
에이전트가 불변식(Task 미기입, Wiki Extractions 누락 등)을 위반하면
hook이 자동 감지하여 다음 세션에 수정 지시를 주입합니다.
wiki 건강성도 `aib wiki lint`로 자동 검사합니다.

덕분에 한 세션에서 끝나는 대화가 아니라,
**프로젝트 차원에서 규칙과 지식이 축적되는 작업 흐름**을 만들 수 있었습니다.

실제로 3개 프로젝트에서 총 80+ 세션을 운용하면서 사용 중입니다.

---

최근에는 **CommandCenter**라는 데스크톱 앱도 만들었습니다.

CLI만으로 배럭을 관리하면서 **파일을 직접 찾아다니는 불편함**,
그리고 **좋은 컨셉들을 까먹는 문제**가 있었기 때문입니다.

CommandCenter에서는:
- 배럭 상태와 에이전트 히스토리를 한눈에 확인
- 설정 파일(SOUL.md, RULES.md, agent.yaml)을 구조화된 UI로 편집
- 완료된 에이전트 작업을 Continue로 이어받기
- 전체 배럭 버전 대시보드 + 일괄 Sync
- 내장 터미널 (xterm.js + portable-pty) — 모든 UI 액션이 터미널에서 실행
- 커맨드 팔레트 (`Cmd+K`) — 배럭 컨텍스트 기반 Quick Commands
- Git 연동 — commit, push, diff, 히스토리 상세 보기
- 세션, 위키, 규칙 통합 검색

Tauri v2 (Rust + React) 기반이라 바이너리 10MB, 메모리 30MB 정도로 가볍습니다.

CLI로 에이전트를 다루는 분이라면 그대로 쓰시면 되고,
**시각적으로 관제하면서 까먹지 않는 것**이 중요하다면 CommandCenter를 추천합니다.

비슷한 고민이 있는 분들에게 참고가 될 수 있어 공유합니다.
